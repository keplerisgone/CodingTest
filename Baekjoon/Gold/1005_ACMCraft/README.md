좋은 질문입니다. 결론부터 말씀드리면
	•	if (--indeg[v] == 0) q.push(v); 는 모든 선행 노드가 처리된 뒤에만 v를 처리하도록 보장하는(=위상정렬의 핵심 불변식) 코드이고,
	•	깊이(최단 단계) 기반 정렬은 어떤 간선 u→v에 대해 depth[u] > depth[v] 또는 depth[u] == depth[v] 같은 상황이 생길 수 있어, 부모가 아직 안 끝났는데 자식을 먼저 처리하는 일이 발생합니다. 그래서 한 번의 순회로는 올바른 DP(최대값 전파 등)가 보장되지 않습니다.

아래에 자세히 설명드리겠습니다.

⸻

1) 왜 --indeg[v] == 0 일 때만 큐에 넣나?

Kahn 위상정렬의 아이디어:
	1.	indeg[x] == 0(들어오는 간선이 없는) 정점들만 처리 가능합니다. → 이들이 “현재 루트”
	2.	하나 꺼내 u를 처리(출력/DP 갱신)하고, u가 내보내는 모든 간선 u→v를 삭제합니다.
구현상 “삭제”는 --indeg[v]로 표현됩니다.
	3.	그 결과 v의 남은 선행 조건이 0개가 되면(--indeg[v] == 0) 비로소 v도 처리 “가능” 상태가 됩니다 → 큐에 push.

불변식: 큐에 들어가는 모든 정점은 남은 선행자가 0입니다.
따라서 큐에서 꺼낼 때 그 노드의 DP 값(예: 최장시간) 은 이미 모든 선행자들로부터 최대값이 충분히 모여 있는 상태입니다.

이 불변식 덕에 한 번의 선형 순회로 올바른 결과가 나옵니다. 만약 사이클이 있으면 어떤 정점도 indegree가 0이 되지 않아서 끝나지 않죠(=DAG 필요).

⸻

2) 왜 “깊이(최단 단계) 정렬”로는 깨지나?

여기서 흔히 말하는 깊이는 보통 멀티 소스 BFS로 얻는 최단 거리(in-degree 0에서의 간선 수 최소)입니다.
그런데 최단 거리는 의존성의 “최대” 조건을 반영하지 못합니다. 구체적으로는 간선 u→v에 대해

\text{depth}(v) \le \text{depth}(u) + 1

만 항상 성립할 뿐, depth(u) < depth(v)가 반드시 성립하진 않습니다. 즉,
	•	depth(u) == depth(v) 이거나
	•	심지어 depth(u) > depth(v) 도 가능합니다(다른 루트에서 v로 더 짧은 경로가 있으면).

그러면 정렬상 v가 u보다 먼저 나올 수 있어, v를 계산할 때 아직 u의 기여가 반영되지 않은 상태가 됩니다.
DP(예: dp[v] = max(dp[v], dp[u] + cost[v]))를 한 번만 돌리면 정답을 놓치게 됩니다.

반례 (간단한 DAG)

루트들: 1, 2   (in-degree 0)
간선:   1 → 4
        2 → 3 → 4

	•	BFS 최단 깊이:
depth[1]=depth[2]=0, depth[3]=1(2→3), depth[4]=1(1→4 경로가 더 짧음)
	•	그런데 의존성은 3 → 4도 있습니다(4의 선행자: 1, 3).

깊이 정렬(오름차순) 은 1,2,3,4 또는 1,2,4,3 같은 순서를 만들 수 있습니다.
두 번째 경우 4가 3보다 먼저 처리되어 3의 기여가 빠진 dp[4] 가 확정되어 버립니다.
반면 위상정렬은 항상 1,2,3,4(또는 2,1,3,4)처럼 모든 선행자를 먼저 처리하므로 안전합니다.

요약: BFS 깊이=“최단 거리”는 의존성 만족과 무관합니다.
위상정렬은 모든 선행 조건이 해소된 순간에만 노드를 꺼내므로 DP가 한 번에 올바르게 수렴합니다.

⸻

3) “그럼 깊이를 ‘최장 거리’로 잡으면 되나요?”

이론적으로 depth[v] = max( depth[u] + 1 ) over u→v(최장 경로 길이)를 쓰면
항상 depth[u] < depth[v] 이라 정렬 안정이 됩니다.
하지만 그 최장 깊이 자체를 계산하려면 위상정렬이 필요합니다(사이클 없는 순서가 먼저 있어야 DP를 안전히 돌릴 수 있음).
즉 닭-달걀 관계입니다. 그래서 실전에서는 Kahn 위상 + DP 로 한 번에 해결합니다.

⸻

4) 코드 관점 요약

for (int v : adj[u]) {
    dp[v] = max(dp[v], dp[u] + cost[v]); // u의 기여를 v에 반영
    if (--indeg[v] == 0) q.push(v);      // 모든 선행자가 처리된 경우에만 v를 큐에
}

	•	이 줄이 간선 제거 시뮬레이션 + 처리 가능성 판정을 동시에 합니다.
	•	불변식 덕분에 큐에서 꺼낼 때 dp[v]는 이미 완전한 값(모든 선행자의 기여가 반영된 최대값)입니다.

⸻

한 줄 정리
	•	--indeg[v]==0 체크는 “선행 조건이 모두 충족된 시점에만 v를 처리”하게 만드는 위상정렬의 핵심입니다.
	•	깊이(최단 단계) 정렬은 선행자가 더 깊거나 같은 깊이일 수 있어 자식이 먼저 처리되는 반례가 반드시 존재합니다.
	•	따라서 의존성 기반 DP(예: ACM Craft)는 Kahn 위상정렬 + DP 로 푸는 것이 정석입니다.
    애초에 문제처럼 부모가 여럿이면 깊이를 자세히 잴 수가 없다
